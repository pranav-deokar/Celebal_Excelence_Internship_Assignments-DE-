# Assignment Insights

## Key Observations

* The database follows a relational model using four interconnected tables: customers, products, orders, and order_items.
* Primary Keys uniquely identify each record, while Foreign Keys maintain relationships between tables.
* Constraints such as PRIMARY KEY, UNIQUE, NOT NULL, CHECK, and FOREIGN KEY help maintain data integrity.
* Filtering techniques using WHERE, BETWEEN, and logical operators efficiently retrieve specific records.
* Aggregate functions such as COUNT, SUM, AVG, MIN, and MAX summarize business data for reporting.
* GROUP BY and HAVING are useful for generating category-wise and status-wise business summaries.
* JOIN operations enable retrieval of meaningful information by combining data from multiple related tables.
* Indexes improve query performance by reducing the time required to search large datasets.
* CASE statements simplify conditional categorization of products and order statuses.
* Transactions ensure that multiple database operations are executed safely while preserving consistency through COMMIT and ROLLBACK.

## Conclusion

This assignment provided practical experience with relational database concepts and SQL query writing. It covered essential SQL operations ranging from basic data retrieval to advanced transaction management, reinforcing the importance of data integrity, optimization, and efficient querying in database systems.
